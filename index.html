Section 01

What is Full Stack Development

When I first heard the term full stack I wasnt really sure what it meant. I thought it just meant knowing a lot about coding in general. But it actually means being able to work on both sides of a web application. The front end is what people see in their browser and the back end is the server and database stuff running behind the scenes.

🖥️ Front End

Everything the user sees and clicks on. The layout the buttons the forms and how things look. We use HTML to structure the page CSS to style it and JavaScript to make it interactive. If a website looks good and works properly in the browser that is all front end work.

⚙️ Back End

The part users dont see. Its the server side logic that handles things like logging in storing data and processing requests. Languages like Node.js Python and PHP are used here. When you sign into a website and it knows who you are that is the back end doing its job.

🗄️ Database

All the data has to be stored somewhere. Databases like MySQL and MongoDB keep everything organised so the back end can read and update it. Without a database most websites wouldnt remember anything.

🛠️ Tools and Version Control

Things like Git for version control so you can track changes and not lose your work. And deployment tools so you can put your site online for people to see. Git has been really useful even just working on my own because I can go back to older versions if something breaks.

Section 02

HTML

HTML stands for HyperText Markup Language and it is the skeleton of any webpage. It tells the browser what content to display and how it should be structured. Every element on a page whether its a heading a paragraph a button or an image is an HTML tag. HTML on its own has no colour or style that is CSSs job.

What are HTML tags?

Tags are the building blocks of HTML. Most tags have an opening and closing version. The browser reads these tags and decides how to display the content inside them.

<h1> Big heading <p> Paragraph <a> Link <img> Image <ul> <li> List <table> Table <form> Form <div> Container <header> Semantic header <nav> Navigation <main> Main content <footer> Footer

Semantic HTML

Using the right tags for the right purpose is called semantic HTML. Instead of putting everything in a div you use tags that describe what the content actually is. This helps screen readers for people with disabilities and it helps Google understand your page better too. I started doing this properly in this module and it made my code look a lot cleaner.

Section 03

CSS

CSS stands for Cascading Style Sheets and it controls how a webpage looks. HTML gives you the structure but without CSS everything is just plain black text on a white page. CSS lets you add colours fonts spacing animations and responsive layouts. I found Flexbox confusing at first but once it clicked it became one of my favourite things to work with.

How CSS works

You select an HTML element and then write properties and values to change how it looks. You can select by tag name by class name or by ID. The cascade part means that if two rules target the same element the more specific one wins. This sounds confusing but once you understand specificity it makes a lot of sense.

Section 04

Bootstrap Framework

Bootstrap is a CSS framework that gives you ready made components and a grid system you can use straight away. You just add the right class names to your HTML and things like responsive columns buttons cards and navbars just work without you having to write loads of CSS yourself. It is built on CSS and JavaScript but you dont need to write those yourself Bootstrap handles it.

The 12 Column Grid

Bootstrap divides every row into 12 invisible columns. You choose how many columns each element takes up. col-6 means half the width. col-4 means one third. You can also add breakpoints like col-md-6 which means half width on medium screens and above but full width on mobile. This is how Bootstrap makes things responsive automatically.

JavaScript

JavaScript is what makes a webpage actually do things. HTML is the structure CSS is the style and JavaScript is the behaviour. Without it everything just sits there and nothing happens when you click or type. JS lets you respond to user actions change what is on screen validate forms do calculations and a lot more. I found JavaScript the hardest part of this module but also the most satisfying when something actually works.
